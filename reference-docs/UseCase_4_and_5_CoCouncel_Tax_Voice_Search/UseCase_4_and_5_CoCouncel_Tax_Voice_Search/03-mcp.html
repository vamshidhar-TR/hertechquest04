MCP

🔌 Connect CoCounsel over MCP

How to reach CoCounsel from an MCP client (Cursor, Claude Desktop).

⚠ Confirm your endpoint first. Get the exact MCP server URL and tool name from your MCP admin. The Tax server, for reference, is https://mcp.accounting.cocounsel.thomsonreuters.com/mcp; the Legal server follows the same pattern. Replace the placeholder <your-cocounsel-mcp-host> below with the confirmed URL.

What MCP is (30 seconds)

MCP (Model Context Protocol) is a standard way for AI clients to call external tools. CoCounsel exposes a research tool over an MCP server. A client connects, lists the tools, and calls one with a question — the server does the research and returns a cited answer. Auth is a one-time OAuth browser login; the token is then cached.

Prerequisite

Node.js + npm on PATH (clients use npx to run a local proxy to the remote server). Check:

node --version

Option A — Cursor

Cursor supports remote HTTP MCP servers natively. Open or create ~/.cursor/mcp.json and add:

{
  "mcpServers": {
    "cocounsel": {
      "url": "https://<your-cocounsel-mcp-host>/mcp",
      "type": "http"
    }
  }
}

Save, then restart Cursor (or reload MCP servers). Complete the browser login when prompted. The CoCounsel tool is then available in Cursor.

Option B — Claude Desktop

Open the config file:

Windows: %APPDATA%\Claude\claude_desktop_config.json
macOS:   ~/Library/Application Support/Claude/claude_desktop_config.json

Add the cocounsel block (keep any existing servers):

{
  "mcpServers": {
    "cocounsel": {
      "command": "npx",
      "args": ["-y", "mcp-remote",
        "https://<your-cocounsel-mcp-host>/mcp",
        "--transport", "http-only"]
    }
  }
}

Fully restart Claude Desktop, complete the browser login, and the tool appears.

Verify

Ask a question in any client:

What is the controlling precedent on fair use in software, and is it still good law?

The response should call the research tool and include citations. Click a citation to confirm it opens. ✓ That means MCP is working.

Troubleshooting

IssueFix
No login prompt appearsConfirm the URL. Restart the client. Disable and re-enable the MCP server.
Login OK but MCP shows disconnectedReturn to the client after the browser confirms. Remove and re-add the server, or sign out of CoCounsel in the browser and retry.
Citation links don't openCheck your browser is authenticated; check whether the client rewrote the link.
Code: connects but times outIncrease the wait() timeout; research calls can take 30–60s+.
