🧩 Tools & Tech — what's in the kit

New to the stack? This is the one-page tour. Each tool below is optional — pick the ones your idea needs. Deeper how-tos are linked where they exist.

Big picture. You build an app (often in an editor like VS Code or Cursor), it turns speech into text with Whisper or Parakeet, sends that text to an LLM / CoCounsel over MCP, and your code lives in a git repo. You don't need all of them.

💻 VS Code

A free, popular code editor. You'll open the project folder here, edit files, and run commands in its built-in terminal. If you're unsure where to start, start here. Download from code.visualstudio.com.

🖱 Cursor

A version of VS Code with an AI assistant baked in — it can explain code, write code, and chat about your project. It also speaks MCP natively, so it's an easy way to reach CoCounsel. See the MCP doc (Option A).

🤖 Codex

OpenAI's AI coding assistant (available as a CLI / in some editors). It's an alternative to Cursor's assistant for generating and editing code. Use whichever AI helper you're comfortable with — none is required to finish the hackathon.

🧠 LLM (Large Language Model)

The technology behind tools like ChatGPT, Claude, and CoCounsel. You give it text (a "prompt") and it responds in natural language. In this kit the LLM does the legal reasoning — your app captures a question, the LLM/CoCounsel answers it with citations.

🔌 MCP (Model Context Protocol)

A shared "plug" standard that lets an app or editor call external tools — here, CoCounsel's research tool. Set it up once and any MCP-aware client can use it. Full setup, the call flow, and troubleshooting are in the MCP doc.

🎙 Whisper (faster-whisper)

A speech-to-text model that turns recorded audio into text. We use faster-whisper because it runs well on a normal CPU — no GPU needed. It's the best default for capturing spoken questions. See the Voice doc.

🦜 Parakeet

NVIDIA's speech-to-text model — higher accuracy than Whisper but it prefers an NVIDIA GPU and downloads a larger model. Use it if you have the hardware and want the best transcription. Also covered in the Voice doc.

🌿 git repo

"Git" tracks every change to your code; a "repo" (repository) is the project folder under that tracking. It lets you save snapshots (commits), undo mistakes, and share with teammates (e.g. via GitHub). Common first commands:

git clone <url> · git status · git add . · git commit -m "message" · git push

Not sure what to use? A minimal path: open the project in VS Code, use Whisper for speech, reach CoCounsel over MCP, and keep your work in a git repo. Add the rest only if your idea calls for it.
