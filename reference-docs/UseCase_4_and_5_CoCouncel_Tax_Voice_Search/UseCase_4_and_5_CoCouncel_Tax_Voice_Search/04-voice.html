🎙 On-device Voice (STT + TTS)

Capture speech and speak answers, fully on your machine. No API keys for the voice layer — models download once and cache locally.

Why on-device? Spoken queries can contain client names, figures, and privileged data. Local STT/TTS keeps the raw audio on your machine. (The transcribed text still goes to CoCounsel's cloud for research — that's its intended destination. Local means private audio, not private end-to-end.)

Speech-to-Text — faster-whisper

A fast CPU Whisper (CTranslate2 + int8). Works without CUDA.

Install

pip install faster-whisper sounddevice soundfile

How it works

  • Load a WhisperModel from faster_whisper, choosing the model size, device="cpu", and compute_type="int8" for the CPU-friendly path.
  • Capture mic audio with sounddevice as 16kHz mono float32, then write it to a WAV (or pass the array directly).
  • Call model.transcribe(...) (try beam_size=5) — it returns an iterable of segments; concatenate their .text for the final transcript.

Model sizes: tiny < base < small < medium < large-v3 (bigger = more accurate, slower on CPU).

Speech-to-Text — Parakeet (NVIDIA)

Install

pip install "parakeet-stream[microphone] @ git+https://github.com/maximerivest/parakeet-stream.git"

How it works

  • Instantiate Parakeet from parakeet_stream — it downloads the model on first run, then caches it.
  • Record a clip from the Microphone device, then pass it to transcribe(...) and read the resulting .text.
  • Best on an NVIDIA GPU; expect a ~600MB model download.

Text-to-Speech — Kokoro

pip install kokoro soundfile   # Linux only: apt-get install espeak-ng

How it works

  • Create a KPipeline from kokoro with a lang_code (e.g. "a" for American English).
  • Feed it text plus a voice and speed; it yields audio chunks you concatenate into one waveform.
  • Play the waveform at 24kHz (e.g. via sounddevice).

Text-to-Speech — Piper

pip install piper-tts
python3 -m piper.download_voices en_US-lessac-medium

How it works

  • Load a voice with PiperVoice.load("<voice>.onnx").
  • Synthesize text to a WAV (e.g. synthesize_wav(...) into an in-memory buffer), then read it back into a waveform + sample rate.
  • Play it — lowest resource of the TTS options, good for short alerts.
Audio format tip. Most STT models expect 16kHz mono WAV. Convert anything with ffmpeg: ffmpeg -i input.mp3 -ar 16000 -ac 1 output.wav

Avoid Claude Desktop voice mode, Mac Caps-Lock dictation, and Claude Code /voice for sensitive input — they stream audio to the cloud, so they are not on-device.
